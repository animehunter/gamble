3 Samplers and Solvers

A sampler is a procedure that takes zero arguments, runs a stochastic program, and produces a single result.

The examples in the following sections use the following function definitions:

> (define (count-heads n)
    (if (zero? n) 0 (+ (if (flip) 1 0) (count-heads (sub1 n)))))
> (define (geom)
    (if (flip) 0 (add1 (geom))))

syntax

(rejection-sampler def/expr ... result-expr maybe-when-clause)

 
maybe-when-clause = 
  | #:when condition-expr
Produces a sampler that, when applied, returns a value of result-expr arising from an execution where condition-expr, if present, was satisfied.

The sampler is implemented using rejection sampling. That is, the def/exprs and result-expr are evaluated. Then condition-expr is evaluated, if present; if it produces #t, the execution is accepted and the value of result-expr is returned; otherwise, the process is repeated.

Examples:

> (define s-or
    (rejection-sampler
      (define A (flip))
      (define B (flip))
      A
      #:when (or A B)))
> (hist (repeat s-or 100))

> (hist (repeat s-or 1000))

> (define rs-count-heads
    (rejection-sampler
     (count-heads 10)))
> (rs-count-heads)

3

> (rs-count-heads)

6

> (hist (repeat rs-count-heads 100))

syntax

(mh-sampler def/expr ... result-expr maybe-when-clause)

 
maybe-when-clause = 
  | #:when condition-expr
Like rejection-sampler, but when applied repeatedly uses a variant of Metropolis-Hastings as described in [Bher].

Examples:

> (define mh-or
    (mh-sampler
      (define A (flip))
      (define B (flip))
      A
      #:when (or A B)))
> (hist (repeat mh-or 100))

> (hist (repeat (lag mh-or 100) 100))

> (define mh-n-flips
    (mh-sampler
     (count-heads 10)))

syntax

(enumerate def/expr ... result-expr
  maybe-when-clause maybe-limit-clause maybe-normalize-clause)
 
maybe-when-clause = 
  | #:when condition-expr
     
maybe-limit-clause = 
  | #:limit limit-expr
     
maybe-normalize-clause = 
  | #:normalize? normalize?-expr
Returns a distribution table of the form (list (list value prob) ...), where value is a value produced by result-expr and prob is its normalized probability given condition-expr. If normalize?-expr is given and evaluates to #f, then the probability is unnormalized instead.

The enumerate form works by exploring all possibilities using the technique described in [EPP]. If limit-expr evaluates to a probability limit, then exploration ceases when the unexplored possibilities have probability less than limit times the sum of the probabilities of the results accepted by condition-expr so far. If limit-expr evalues to #f, then exploration ceases only when all paths have been explored; if any path is infinite, then enumerate fails to terminate.

Only discrete and integer-valued ERPs can be used with enumerate, and infinite-range ERPs (such as geometric) require the use of #:limit to enforce termination.

Examples:

> (enumerate
    (define A (flip))
    (define B (flip))
    A
    #:when (or A B))

'((#f 0.3333333333333333) (#t 0.6666666666666666))

> (enumerate

Use #:normalize? #f to get the result probabilities without normalizing by the acceptance rate:

> (enumerate
   (define (drop-coin?) (flip 0.9))
   (define (drunk-flips n)
     (cond [(zero? n)
            #t]
           [(drop-coin?)
            'failed]
           [else
            (and (flip) (drunk-flips (sub1 n)))]))
   (define A (drunk-flips 10))
   (eq? A #t)
   #:when (not (eq? A 'failed))
   #:normalize? #f
   ; Default limit is too high to detect #t case
   #:limit #f)

'((#f 0.05263157894736328) (#t 9.765624999999978e-14))
